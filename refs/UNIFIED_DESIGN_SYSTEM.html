07
Guidelines

Design Principles

01
Attention Is the Interface
Every interaction is an attention event. Navigating, hovering, dwelling — these are the system's heartbeat. Heat indicators make collective attention visible. Using the system generates value.
02
Fractal Depth, Never Flat
The artifact tree nests infinitely. Navigation feels like zooming — deeper into sub-spaces. The same component renders at every level. Nothing is a "special page."
03
Quiet Until Relevant
The interface defaults to stillness. Information surfaces when it matters — a peer joins, a quest gets claimed, an exchange completes. Content speaks; chrome listens.
08
Do's & Don'ts

Cultivate & Compost

✓ Do
Use var(--accent-primary) for interactive elements
✕ Don't
Hard-code hex values that break across biomes
✓ Do
Member colors for identity — avatar rings, sender names
✕ Don't
Member colors as decoration or status indicators
✓ Do
Respect the spacing scale — --space-* tokens
✕ Don't
Arbitrary pixel values for margins and padding
✓ Do
Let heat drive warmth via --heat
✕ Don't
Animate elements with no attention data behind them
✓ Do
Design offline-first — stale data with sync indicators
✕ Don't
Blank screens when connectivity drops
✓ Do
Test every component across all five biomes
✕ Don't
Design for one theme, assume the rest adapt
